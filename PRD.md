# Orbit（新产品）PRD

> 面向个人用户的 **原生 macOS SSH 管理终端**：把“连接（SSH）+ 文件（SFTP）+ 状态（监控）+ 结构化管理（分组/多 Tab）”收拢到一个轻量、离线优先、本地加密的工具里。  
> 参考产品：DartShell（macOS 一体化远程运维客户端）、Nexterm（结构化管理/监控/脚本化思路）。  
> 现有雏形：Orbit 仓库 `feature/macos-optimization` 分支（SwiftUI/AppKit + SwiftTerm/Swift Charts + Rust FFI），README 中功能已全部实现。

---

## 1. 背景与问题陈述

### 1.1 背景
你目前使用的多种工具组合（SSH 客户端、SFTP、监控、分组管理等）存在体验割裂、切换成本高、配置分散/不透明、效率低等问题。因此需要做一个“个人向、免费、够用且快”的替代品。

### 1.2 目标用户痛点（来自竞品启发）
- **减少应用切换**：DartShell 强调 integrated workflow；把常用远程操作放在一个 App 内完成。
- **结构化管理资产**：Nexterm 强调 folders/tabs/snippets，让“多服务器、多会话”可控。
- **安全与隐私**：离线优先、本地加密存储、无需登录（DartShell 方向）。
- **批量/并行效率**：分组、广播、分屏、连接复用，适用于多台服务器的日常维护。

## 2. 产品愿景与定位

### 2.1 产品愿景
做一款“打开就能用、离线也能用、越用越顺手”的 macOS 桌面 SSH 管理终端，让个人运维/开发的远程日常操作更集中、更快、更安全。

### 2.2 产品定位（一句话）
**Orbit = 面向个人的轻量级远程运维工作台（SSH + SFTP + 监控 + 分组批量）**。

### 2.3 非目标（明确不做/暂不做）
（避免走向 Nexterm 那种“平台化”）
- 不做：多租户/组织、SSO、权限体系、审计报表（除非后续有明确需求）。
- 不做：云端账号体系、数据云同步（默认保持离线优先；后续可选 iCloud/本地文件同步方案）。
- 不做：企业级堡垒机替代。

## 3. 目标用户与使用场景

### 3.1 目标用户画像
1) **独立开发者/个人站长**
- 需要频繁 SSH 到多台 VPS / 裸金属
- 常用：查看日志、重启服务、拉取/上传文件、看资源占用

2) **小团队中的“兼职运维”**
- 服务器数量不算太多，但需要分组管理（prod/staging/dev）
- 强依赖“快连 + 批量操作 + 稳定”

### 3.2 核心场景（Top 5）
1. 快速连接某台服务器，打开终端开始操作
2. 同时维护多台服务器：多 Tab 并行观察与操作（可选：未来加入分屏）
3. 上传配置/下载日志：在 SFTP 内完成基本文件操作
4. 通过跳板机访问内网机器
5. 诊断性能问题：查看 CPU/内存/磁盘趋势 + 进程/资源概览（按 MVP 能力逐步）

  - 跳板机代理（自动建立 TCP 转发隧道）
  - AES-256-GCM 加密存储密码/私钥，**绑定本机**（隐私优先）
- 工作台体验
  - 多 Tab 管理（终端 / SFTP / 监控）

- 配置导入/导出（JSON）：迁移/备份/共享

- 命令片段（Snippets）：保存常用命令，一键执行（按分组/标签组织）
- 多服务器批量执行：选中多台服务器执行同一命令，结果汇总展示（含失败重试/超时）
- 本地端口转发管理（ssh -L）：可视化规则 + 一键开关
- 连接日志：记录每次连接时间、时长、流量与错误原因（便于复盘）

### 4.3 V2（可选增强，按你的真实痛点再决定）
- 远程编辑（轻量）：下载到临时目录 → 本地编辑 → 自动回传（带差异提示）

- 个性化设置：主题/字体/光标/快捷键（从“够用”走向“顺手”）
- 本地同步方案（仍不引入账号）：导入/导出 + 可选 iCloud Drive 路径
- AI 助手（可选）：命令生成/风险审计/日志诊断/SQLite 自然语言转 SQL（**默认支持 OpenAI 与 Anthropic 协议，用户自填 API Key，本地加密存储**）

## 5. 竞品对标（提炼“该学什么/不学什么”）

### 5.1 DartShell 值得借鉴
- 一体化工作流：连接、传输、转发、跳板机、分屏、广播等让“连续工作”更顺
- 离线优先 + 本地存储 + 不登录：对个人用户非常友好
- 复杂网络支持：代理/跳板机/转发（Orbit 优先补齐“够用的网络能力”）

### 5.2 Nexterm 值得借鉴
- 结构化管理（folders/tabs）与操作组织方式
- Snippets / Scripts：把重复操作标准化，提高效率
- 监控面板：把“看状态”放在连接旁边，而不是另开工具

### 5.3 明确不走的方向（保持个人轻量）
- 团队组织/SSO/多用户权限（平台化成本高，且与个人免费目标冲突）

### 5.4 Orbit 的差异化（新增）
> 与 DartShell / Nexterm 的“功能堆叠或平台化”不同，Orbit 的差异化聚焦在：**原生体验 + 极简工作流 + 可控的自动化**。

1) **原生 macOS 体验与性能（主打）**
- SwiftUI + AppKit 原生交互、窗口/菜单/快捷键更贴合 macOS 使用习惯
- SwiftTerm + Metal GPU 渲染：在长时间滚动、大量输出时更稳、更顺
- Swift Charts：监控趋势图不依赖 Web/前端图表框架，降低资源占用

2) **“离线优先 + 隐私优先”的默认选择**
- 无账号体系、不上传数据；凭据 AES-256-GCM 加密存储
- **绑定本机**：默认更安全（同时在 PRD 里规划“可迁移的导入/导出”来平衡体验）

3) **围绕“个人日常”的高频链路做极致**
- 三件事闭环：连上（SSH）→ 看状态（Monitor）→ 传文件（SFTP）
- 在“管理多台服务器”上投入：搜索/分组/批量执行/片段库/连接日志（这些比“多协议”更常用）

4) **开源可控（MIT）+ 可演进的架构**
- Rust 核心能力通过静态库 + C ABI 暴露：稳定、可测试、未来可复用
- UI 维持原生，避免“Electron/Tauri 的系统感缺失”和复杂前端依赖堆叠

## 6. 信息架构与关键页面

### 6.1 导航结构（建议）
- 左侧 Sidebar
  - 搜索框
  - 分组（可折叠）
  - 服务器条目（状态点：在线/离线/最近连接）
  - 右键菜单：连接 / 打开 SFTP / 打开监控 / 编辑 / 删除 / 复制连接信息
- 主区域 Tab Bar
  - Terminal Tabs（可选：未来分屏）
  - SFTP Tabs
  - Monitor Tabs
  - DB Tabs（SQLite 面板）
- 全局工具区（顶部/右上）
  - 新建服务器/分组/凭据组
  - Snippets（V1）
  - AI（可选）：Ask AI / Review / Diagnose
  - 转发规则（V1）
  - 设置（主题、快捷键、刷新频率、日志、导入导出）

### 6.2 关键交互（必须顺滑）
- 快捷键：快速搜索服务器、快速新开终端、Tab 切换焦点、关闭 Tab（可选：未来分屏快捷键）
- 懒加载：不打开就不建立连接/不拉监控
- 连接状态反馈：连接中、失败原因（超时/认证失败/不可达）

### 6.3 数据库模块（SQLite 面板）布局与交互（新增）
> 目标：满足“内网机器上的 SQLite，只能通过跳板机访问”的场景；提供足够好用的**页面展示 + 增删改查**能力（不追求 DB IDE 的全部能力）。

#### 布局（3 区域）
- **左侧：表列表**
  - 展示当前 SQLite DB 的表/视图列表（支持搜索）
  - 点击表：中间可一键 `SELECT * LIMIT N` 预览（可选），右侧展示表结构/DDL
- **中间：上下分区**
  - 上：SQL 输入框（多行，支持快捷键执行：⌘+Enter）
  - 下：执行结果输出（表格渲染、行数、耗时、错误信息、复制/导出）
- **右侧：表结构**
  - 展示选中表的 columns 列表与类型/约束
  - 或直接展示 DDL（`.schema table_name` 的输出）

#### 最小交互约束（避免“卡死/输出爆炸”）
- 默认强制添加 `LIMIT`（例如 200），允许用户手动调整
- 单次输出行数/字节数上限（超出给出提示：请加 LIMIT 或导出到文件）
- 运行超时（例如 10s/30s 可配置），超时可取消
- 默认只读模式（可在设置中显式开启写入；开启前二次确认）

## 7. 功能需求（Functional Requirements）

### 7.1 服务器与凭据
- FR-1：支持创建/编辑/删除服务器配置（host/port/user/备注/分组/标签）
- FR-2：支持凭据组（密码/私钥/Passphrase/用户名复用）
- FR-3：保存前一键测试连接，并将结果明确展示（耗时、错误原因）
- FR-4：最近连接记录（最近 N 条，可一键重连）

### 7.2 SSH 终端
- FR-5：支持标准终端体验（xterm-256color、复制粘贴、滚动回看、搜索）
- FR-6：多 Tab 管理（终端 / SFTP / 监控），Tab 命名、关闭确认、异常状态提示
- FR-7：命令片段（Snippets）（V1）：一键插入/一键执行，支持按分组组织
- FR-8：多服务器批量执行（V1）：支持选择服务器集合、并发上限、超时、结果聚合与导出

### 7.3 SFTP
- FR-9：远程目录浏览、下载、上传、删除、新建文件夹
- FR-10：传输队列与状态（进度、速度、失败原因、重试、并发）（V1）

### 7.4 跳板机/转发
- FR-11：服务器可选择“通过跳板机连接”
- FR-12：本地端口转发规则管理（V1）：新增/删除/启停/冲突检测（端口占用）

### 7.5 监控
- FR-13：展示 CPU/内存/磁盘使用率与趋势
- FR-14：展示网络流量（会话级别）与基本系统信息（可扩展）

### 7.6 安全与隐私
- FR-15：敏感信息本地加密存储（默认开启）
- FR-16：设备绑定策略的说明与可迁移路径（导出不含明文凭据/或通过二次加密导出）
- FR-17：支持锁屏策略（可选：Touch ID / 应用锁）

### 7.7 数据库（SQLite Remote Console）（新增）
> 关键点：SQLite 不走“端口转发连接”，而是通过 Orbit 已有 SSH/跳板机链路，在远端执行 `sqlite3` 命令（exec channel），把 stdout/stderr 拉回渲染。

- FR-18：支持创建/管理 SQLite 数据库配置（Database Profile）
  - 关联服务器（直连或跳板机链路复用现有 Server）
  - 远端 DB 文件路径（例如 `/var/lib/app/app.db`）
  - 可选：只读模式开关（默认开启）
- FR-19：打开 DB 面板后，自动加载表列表（左侧）
  - 实现方式：执行 `sqlite3 <db> ".tables"`（或查询 sqlite_master）
- FR-20：SQL 执行（中间上方输入框）
  - 支持多行 SQL（至少支持单条；多条可后续）
  - 支持 ⌘+Enter 执行、按钮执行、清空
  - 支持取消执行（中断 channel / 标记结果无效）
- FR-21：结果渲染（中间下方）
  - 表格展示（列名、行）
  - 展示元信息：耗时、返回行数、错误信息
  - 支持复制选中/复制为 TSV/CSV（最小可用）
  - 支持导出结果（CSV/JSON，优先 CSV）
- FR-22：表结构/DDL（右侧）
  - 点击表时，右侧展示 `.schema <table>` 输出（或 `pragma table_info` + 约束信息）
- FR-23：可用性检测与错误提示
  - 远端无 `sqlite3`：提示“需要安装 sqlite3”，并展示对应系统的安装建议（可复制命令）
  - 权限不足/文件不存在：明确错误原因，提示路径检查或权限提升
- FR-24：大结果防护
  - 默认在执行前提示/自动注入 `LIMIT N`（可配置 N）
  - 结果大小超过阈值：提示用户加 LIMIT 或导出到远端文件后再下载（可选增强）

### 7.8 AI 助手（可选，默认支持 OpenAI / Anthropic 协议）（新增）
> 产品原则：**AI 只给建议，不自动执行**；**默认最小化外发内容**；不做账号体系，用户自填 Key。

#### Provider 与配置
- FR-25：Settings 支持选择 AI Provider：OpenAI / Anthropic
- FR-26：支持配置项
  - API Key（本地加密存储）
  - 模型名（手动输入即可，避免强耦合某个型号）
  - 可选 Base URL（仅 OpenAI：用于兼容 OpenAI 协议的自建/第三方网关）
  - 一键连通性测试（成功/失败原因提示）

#### 核心能力（最小 MVP）
- FR-27：Terminal Ask AI（自然语言 → 命令建议）
  - 输出：建议命令 + 解释 + “复制/插入/执行”
- FR-28：Terminal Review（风险审计）
  - 识别高危命令（rm/iptables/chmod/覆盖配置等）并强制二次确认
  - 输出：风险原因 + 更安全替代命令 + 回滚建议
- FR-29：SQLite NL → SQL
  - 用户输入需求，AI 生成 SQL（默认加 LIMIT）
  - 支持 Explain SQL / Fix Error（根据错误信息建议修正）

#### 隐私与安全约束（强制）
- FR-30：默认仅发送“用户选中的文本 + 最小必要上下文”（如 OS、shell、当前目录、目标服务器别名），不上传完整日志/历史
- FR-31：禁止自动执行（必须由用户点击确认执行）
- FR-32：可一键关闭 AI（全局开关）

## 8. 非功能需求（NFR）

- NFR-1 性能：启动 < 2s（冷启动），常用操作（打开连接、切换 tab）无明显卡顿
- NFR-2 稳定性：长连接稳定（≥ 8 小时不掉线为目标），网络抖动可自动重连（可选）
- NFR-3 安全：本地加密、最小权限；日志不泄露敏感字段
- NFR-4 平台策略：macOS 14+ 优先（先把体验做到极致）；其他平台仅在不影响 macOS 体验前提下再评估
- NFR-5 可维护性：连接层/传输层/存储层解耦，便于后续加协议或替换库

## 10. 里程碑与迭代计划（面向个人开发节奏）

### M0（已完成：MVP 基线）
- `feature/macos-optimization` 分支 README 列出的功能已实现并可用（SSH/SFTP/监控/跳板机/分组/加密/多 Tab）
- 技术路线明确：原生 SwiftUI + Rust FFI（为“性能与系统感”服务）

### M1（差异化第一阶段：管理与迁移体验）
目标：让 Orbit 在“个人多服务器管理”上明显优于常见 SSH 客户端。
- Sidebar 搜索过滤（服务器/分组/标签）
- 配置导入/导出（JSON）：支持备份与迁移
- SQLite 数据库面板（远程执行）：表列表 + SQL 执行 + 结果展示 + DDL（用于内网 DB/跳板机场景）
- 设置项补齐：刷新频率、默认 shell/环境、主题/字体（先做最小可用）
验收标准：
- 迁移：换设备/重装可 5 分钟恢复服务器列表（凭据可选择重新输入）
- 管理：服务器数量上来（例如 50+）仍然好找、好用

### M2（差异化第二阶段：自动化与批量效率）
目标：把 DartShell/Nexterm 的“效率点”做成更适合个人的轻量版本。
- 命令片段（Snippets）+ 一键执行
- 多服务器批量执行（结果聚合、失败重试、超时控制）
- 本地端口转发管理（ssh -L）：UI 化、一键开关
（可选增强）：
- AI 助手 MVP：Ask AI / Review（风险审计）/ SQLite NL→SQL（默认 OpenAI/Anthropic 协议）
验收标准：
- 你常见的 3~5 个重复操作可以“一键化”（不再手动复制粘贴）

### M3（差异化第三阶段：可观测与复盘）
- 连接日志（时间、时长、流量、错误原因）
- 终端录制/回放（可选）：用于问题定位与经验沉淀

## 11. 成功指标（个人产品的可度量目标）

> 不以商业收入为目标，指标以“替代成功/效率提升/稳定性”为主。

- 使用替代率：你日常远程操作中，Orbit 覆盖 ≥ 90%
- 日均切换次数：从 “多工具切换” 降到 “Orbit 内完成”为主（主观记录即可）
- 稳定性：连续 7 天使用无阻断级 bug（阻断=无法连接/数据丢失/崩溃频繁）
- 性能：打开/切换 tab、分屏输入无明显延迟；SFTP 传输稳定

## 12. 风险与对策

| 风险 | 影响 | 对策 |
|---|---|---|
| SSH 库兼容性/稳定性（不同 key/算法/服务器实现） | 连接失败、口碑差 | 做“连接诊断”与详细错误提示；建立回归用例；必要时可切换/封装 SSH 实现 |
| 监控与 SFTP 共用连接导致的资源竞争 | 卡顿/掉线 | 连接池限流 + 超时策略；必要时拆分连接（可配置） |
| 本地加密/存储迁移导致数据损坏 | 数据丢失 | 强制导出备份；版本化 schema；提供“重置但保留备份文件” |
| 范围膨胀（多协议、平台化） | 失去轻量与个人定位 | 明确非目标；每次新增功能必须满足“你强需求 + 不显著增加复杂度” |

## 13. 开发建议（与当前 Orbit 技术栈一致）

- UI：SwiftUI + AppKit（原生 macOS 体验优先）
- 终端：SwiftTerm（xterm-256color PTY，Metal 渲染）
- 图表：Swift Charts（监控趋势）
- 核心能力：Rust（ssh2/rusqlite/aes-gcm）编译为静态库，通过 cbindgen 生成 C ABI 给 Swift 调用
- 数据：SQLite（bundled），本地存储；明确数据路径与备份策略
- 发布：优先 macOS；后续补签名/公证（Notarization）与自动更新策略（如果你真的要长期自用，也能省心）
